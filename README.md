### AI 기반 에러 분석 파이프라인 - 초기
---
Spring Boot 기반 애플리케이션에서 발생하는 에러를 SDK를 통해 자동 수집하고, 중앙 Error Server에서 저장·그룹화·분석·알림까지 수행하는 통합 에러 모니터링 플랫폼입니다.

분산된 서비스 환경에서 발생하는 에러를 하나의 서버에서 통합 관리할 수 있도록 설계하였으며, Slack 실시간 알림과 AI 기반 원인 분석 기능을 통해 장애 대응 과정을 자동화하는 것을 목표로 하였습니다.

또한, 스택 트레이스와 GitHub 소스코드를 기반으로 AI가 원인 및 해결 방안을 분석하고, 향후에는 자동 코드 수정 PR 생성 기능까지 확장하는 것을 목표로 하고 있습니다.<br><br>

### 핵심 기능
---
- SDK 기반 자동 에러 수집

- Filter / MVC / Logback 기반 다중 예외 감지

- Fingerprint 기반 에러 그룹화

- 비동기 전송 및 로컬 백업 지원

- Slack 실시간 알림

- 프로젝트별 에러 대시보드

- AI 기반 원인 분석 및 해결 방안 생성

- GitHub 소스코드 조회 기반 분석

- 향후 AI 자동 PR 생성 예정<br><br>

### Architecture 
---
<pre>
[각 서비스 (Spring Boot)]
        │
        │ (SDK가 에러 감지)
        ▼
[error-monitor-sdk]
        │
        │ ErrorReport (JSON)
        ▼
[error-monitor-server]
        │
        ├─ 1. DB 저장 (ErrorEvent 저장 및 그룹화)
        │
        ├─ 2. Slack 알림 (신규/재발 에러 발생 알림)
        │
        ├─ 3. AI 분석 수행 (비동기)
        │      ├─ 스택 트레이스 정제
        │      ├─ GitHub 코드 조회
        │      └─ 원인 및 해결 방법 도출
        │
        ├─ 4. 분석 결과 DB 저장
        │
        ├─ 5. 대시보드 출력
        │      ├─ 에러 목록
        │      ├─ 상세 스택 트레이스
        │      └─ AI 분석 결과
        │
        └─ 6. Slack 추가 알림
               ├─ 근본 원인 요약
               └─ 해결 방법 3가지 제공
</pre><br><br>

### 작동 흐름
---
<pre>
예외 발생
→ SDK 자동 감지
→ ErrorEvent 생성
→ fingerprint 생성
→ 비동기 전송
→ Error Server 저장
→ 에러 그룹화
→ Slack 알림
→ AI 분석 수행
→ Dashboard 표시
</pre><br><br>

### AI 분석 흐름
---
<pre>
Error 발생
→ StackTrace 수집
→ StackTrace 정제
→ GitHub 코드 조회
→ AI 분석 요청
→ 근본 원인 분석
→ 해결 방안 생성
→ Dashboard / Slack 표시
</pre><br><br>

### 핵심 원칙
---
1. SDK는 메인 애플리케이션을 방해하지 않아야 한다. (비동기 처리 + 실패 시 무시)
   
2. 같은 원인의 에러는 그룹핑하여 처리한다.
   
3. 민감 정보는 SDK 단계에서 반드시 마스킹한다.
   
4. 모든 에러가 아닌, 의미 있는 이벤트만 알림으로 전달한다.
   
5. 에러 분석 시 과거 해결 사례를 재사용한다.<br><br>


에러 발생 → SDK 수집 → 서버 저장 → 그룹핑 → 알림 및 AI 분석 → AI 기반 조회<br><br>
